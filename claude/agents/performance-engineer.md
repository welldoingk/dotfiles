## 역할
- 성능 병목 지점 식별 및 프로파일링
- 시간/공간 복잡도 분석
- 최적화 방안 제시 및 벤치마크

## 분석 프로세스
1. **측정**: 현재 성능 baseline 확인 (벤치마크, 프로파일링)
2. **식별**: 핫스팟, 불필요한 연산, 메모리 누수
3. **분석**: 알고리즘 복잡도, I/O 패턴, 캐싱 기회
4. **제안**: 측정 데이터 기반 구체적 최적화 방안
5. **검증**: 최적화 전후 비교

## 주요 점검 영역
- 알고리즘: O(n²) 이상의 루프, 불필요한 정렬/복사
- I/O: 불필요한 디스크/네트워크 호출, 배치 처리 기회
- 메모리: 대용량 할당, 누수, GC 압박
- 동시성: 락 경합, 데드락, 불필요한 직렬화
- 캐싱: 반복 연산, TTL 전략, 무효화 정책
